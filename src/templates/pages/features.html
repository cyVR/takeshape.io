Developers

Tech Stack

TakeShape is built on rock solid technologies to keep you up and running for the long term.

We run on an AWS serverless architecture. We use triple-redundant DynamoDB and S3 to store your content and assets. We want your content to be available lickey quick so we front everything with a CDN. Our current unit test code coverage is 65% and rising.

  • Multiple Environments

    Separate environments for developer, staging, and production. Move seamlessly between them with automatic migrations.

  • Multiple Projects

    A single login for all of your TakeShape projects.

  • API

    Supports both high-throughput reads and writes. REST and GraphQL endpoints.

  • Hosting

    We host your site with options for: custom domains, HTTPS, case-insensitive paths, URL redirects, and atomic deploys.

  • Internationalization (I18N)

    Add support for any language you wish.

  • Search

    Faceted search, geo search, full text search.

  • CLI

    Use the ```ts``` command line tool to create and deploy project.

  • Static Site Generator

    Built-in static site generator with a flexible asset pipeline, local image proxy, and easy-to-use templating language (Nunjucks).

Modelers

  • Content Agnostic

    It's your content, there's no single right way to model it.

    TakeShape allows you to build your content model to fit anything the designers throw at you.

  • Content Modeling GUI

    We’ve made modeling a drag and drop experience.

    We want it to be as easy as possible for technical and non-technical users to model content in TakeShape. If you’re a product manager we want you to be able to model.

  • Schemas

    Schemas are independent of form definitions

    They also have the ability to validate and hold defaults. You can import and export them too!
